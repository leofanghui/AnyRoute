# Minimal i18n Guide

The minimal source profile keeps the existing `next-intl` runtime shape, but only
ships the dashboard locales needed by the trimmed UI:

| Locale  | Message file                   |
| ------- | ------------------------------ |
| `en`    | `src/i18n/messages/en.json`    |
| `zh-CN` | `src/i18n/messages/zh-CN.json` |

`config/i18n.json` remains the source of truth for the active locale list.
`src/i18n/config.ts` exposes that list to the app, and
`src/i18n/request.ts` loads the selected JSON file with English fallback support.

## Runtime Flow

1. The language selector writes the `NEXT_LOCALE` cookie.
2. `src/i18n/request.ts` validates the requested locale against `config/i18n.json`.
3. The matching message file is loaded from `src/i18n/messages/`.
4. Missing `zh-CN` keys fall back to `en`.

## Restoring a Locale

To re-add a removed language later, restore its `src/i18n/messages/{locale}.json`
file and add the matching entry back to `config/i18n.json`.
